# Anime Recommendation System

A full-stack anime recommendation system built on the [Anime Recommendation Database 2020](https://www.kaggle.com/datasets/hernan4444/anime-recommendation-database-2020) from Kaggle. The project covers the complete pipeline: data exploration, preprocessing, model development, evaluation, and a live web-based proof of concept.

## Repository Structure

```
├── EDA.ipynb                          # Data exploration, cleaning, preprocessing
├── modeling/
│   ├── modeling.ipynb                 # Model development, evaluation, comparison

├── scripts/

│   ├── build_anime_with_synopsis.py   # Filter synopsis data for catalog

│   │   ├── catalog.json               # Anime metadata catalog
│   │   ├── recommendations.json       # Pre-computed recommendations per user
│   │   └── posters.json               # Poster URL overrides
│   └── src/                           # React + TypeScript source
├── requirements.txt                   # Python dependencies
└── .gitignore
```

## Pipeline Overview

### 1. Data Engineering (`EDA.ipynb`)

Exploratory data analysis and preprocessing on two datasets:
- **anime.csv** (~17,562 anime entries, 35 columns)
- **animelist.csv** (~109M user-anime interactions)

Key operations:
- Chunked CSV reading for the 2 GB animelist file
- Missing value analysis (detects "Unknown" placeholders across 7+ columns)
- 11 EDA visualizations on anime metadata (type, score, genre, episodes, source, popularity, temporal trends, correlations)
- 6 EDA visualizations on user interactions (rating distribution, watching status, user activity, anime popularity)
- Combined analysis (user vs MAL scores, cold-start analysis, user bias)
- Preprocessing: genre one-hot encoding (41 columns), source grouping, feature engineering (Fav_ratio), cold-start filtering
- Exports 4 clean datasets to `processed_data/`

**Key findings:**
- 98% user-item matrix sparsity — favors matrix factorization over memory-based methods
- 42.9% of interactions are unrated (implicit feedback)
- Long-tail popularity distribution — small fraction of anime drives most engagement
- 66% of users rate above the global mean — user bias normalization recommended

### 2. Modeling (`modeling/modeling.ipynb`)

Implements and evaluates 13 recommendation approaches:

| Category | Models | Best RMSE |
|----------|--------|-----------|
| Non-personalized | Popular, Random | 1.69 |
| Collaborative filtering | SVD, KNN Item-based, KNN User-based | 1.23 (SVD) |
| Content-based | Metadata, BoW, TF-IDF, BERT, NER | 1.45 (TF-IDF) |
| Hybrid | Weighted (SVD + TF-IDF + Metadata + Popular) | 1.65 |
| Exploration | Thompson Sampling bandit | N/A |

**Evaluation metrics:** RMSE, MAE, Precision@10, Recall@10, NDCG@10, Coverage

**Key results:**
- SVD achieves the best accuracy (RMSE 1.23) but low coverage (1.45%)
- Hybrid model achieves the best ranking metrics (Precision@10 0.84, NDCG@10 0.98) with improved coverage (2.93%)
- Content-based models help address the cold-start problem
- Hyperparameter tuning via GridSearchCV improves SVD RMSE from 1.235 to 1.222

### 3. Web Application (`web/`)

A Netflix-style React frontend that showcases the recommendation system as a proof of concept:

- **Hero spotlight** — SVD-recommended anime with dynamic poster-extracted color gradients
- **Recommended for you** — Hybrid model (SVD + Popular) personalized picks
- **Crowd-pleasers** — Global popular recommendations (non-personalized baseline)
- **Surprise picks** — Random baseline for exploration
- **Trending / Highest rated** — Catalog-based rows sorted by members and score
- **Search and filtering** — Text search, type and genre filters
- **User profiles** — Switch between pre-computed user profiles
- **Save and rate** — Bookmark and rate anime (persisted in localStorage)

The frontend consumes pre-computed JSON files generated by the modeling pipeline, demonstrating how the recommendation system would function in production with static data.

## Setup

### Prerequisites

- Python 3.10+
- Node.js 18+ (for the web frontend)

### 1. Install Python dependencies

```bash
pip install -r requirements.txt
```

### 2. Download the raw data

Download from [Kaggle](https://www.kaggle.com/datasets/hernan4444/anime-recommendation-database-2020) and place in the project root:
- `anime.csv` (~6 MB)
- `animelist.csv` (~2 GB)

### 3. Run the data pipeline

```bash
# Step 1: EDA and preprocessing (generates processed_data/)
# Open EDA.ipynb in Jupyter and run all cells (~15 min)

# Step 2: Run modeling notebook
# Open modeling/modeling.ipynb and run all cells

# Step 3: Generate web data (optional — pre-built JSONs are in the repo)
python scripts/build_web_catalog.py
python modeling/export_static_recommendations.py
```

### 4. Run the web frontend

```bash
cd web
npm install
npm run dev
```

Open http://localhost:5173 to view the recommendation UI.
